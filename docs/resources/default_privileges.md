# Resource (postgresql_default_privileges)

Manages PostgreSQL default privileges using `ALTER DEFAULT PRIVILEGES`. Default privileges define the privileges that will be applied to objects created in the future by a specific role.

~> **Note:** The import ID format uses `/` as a separator: `role/database/schema/object_type/grantee`. For example: `terraform import postgresql_default_privileges.tables owner_role/my_database/public/table/app_role`.

## Example Usage

```terraform
# Default table privileges: read-only for future tables
# Required: owner, role, database, object_type, privileges
resource "postgresql_default_privileges" "readonly_tables" {
  owner       = "app_owner"        # Required: role that creates objects
  role        = "readonly"         # Required: role that receives privileges
  database    = "my_application"   # Required
  schema      = "public"           # Optional: if omitted, applies database-wide
  object_type = "table"            # Required: table, sequence, function, or type
  privileges  = ["SELECT"]         # Required
}

# Default table privileges: full CRUD for the app user
resource "postgresql_default_privileges" "app_tables" {
  owner       = "app_owner"
  role        = "app_user"
  database    = "my_application"
  schema      = "public"
  object_type = "table"
  privileges  = ["SELECT", "INSERT", "UPDATE", "DELETE"]
}

# Default sequence privileges (needed for serial/identity columns)
```

## Schema

### Required

- `database` (String) Target database where default privileges are applied.
- `object_type` (String) Object type for which default privileges are set. Valid values: table, sequence, function, type.
- `owner` (String) Role that owns future objects.
- `privileges` (Set of String) Set of privileges to grant as default privileges.
- `role` (String) Grantee role that will receive the default privileges.

### Optional

- `schema` (String) Target schema where default privileges are applied. If omitted, defaults apply database-wide.
- `timeouts` (Block, Optional) (see [below for nested schema](#nestedblock--timeouts))
