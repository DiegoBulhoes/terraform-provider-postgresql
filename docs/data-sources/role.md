# Data Source (postgresql_role)

Reads information about an existing PostgreSQL role.

## Example Usage

```terraform
# Look up an existing role
data "postgresql_role" "admin" {
  name = "postgres"
}

output "admin_oid" {
  value = data.postgresql_role.admin.oid
}

output "is_superuser" {
  value = data.postgresql_role.admin.superuser
}

# Use a role data source to conditionally create resources
data "postgresql_role" "app" {
  name = "app_user"
}

resource "postgresql_grant" "app_tables" {
  role        = data.postgresql_role.app.name
  database    = "my_application"
  schema      = "public"
  object_type = "table"
  privileges  = data.postgresql_role.app.superuser ? [] : ["SELECT", "INSERT"]
}

# Check role memberships
data "postgresql_role" "developer" {
  name = "developer"
}

output "developer_memberships" {
  value = data.postgresql_role.developer.roles
}

output "developer_can_login" {
  value = data.postgresql_role.developer.login
}

output "developer_can_create_db" {
  value = data.postgresql_role.developer.create_database
}
```

## Schema

### Required

- `name` (String) The name of the role.

### Read-Only

- `connection_limit` (Number) Connection limit for the role. -1 means no limit.
- `create_database` (Boolean) Whether the role can create databases.
- `create_role` (Boolean) Whether the role can create other roles.
- `login` (Boolean) Whether the role can log in.
- `oid` (Number) The OID of the role.
- `replication` (Boolean) Whether the role can initiate streaming replication.
- `roles` (List of String) List of roles that this role is a member of.
- `superuser` (Boolean) Whether the role is a superuser.
- `valid_until` (String) Password expiry time. Empty if no expiry.
